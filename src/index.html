Buttons

Default

To create a button, add the btn classname to any a or button element. This base class must be used alongside all modifiers. The default button is also a secondary button. As a note, buttons widths are not fixed—they are sized relative to their default padding of interior elements (with the exception of a width-modifying class).

Disabled

To indicate that a button is disabled, add the btn--disabled class alongside btn. It can be used along any colored button and will reduce the opacity of that base color.

Ordinality

To indicate the a button represents a primary action, add the btn--primary class alongside btn. Secondary buttons do not need a modifying class because the style is the same as the default button.

Accent

An accent button provides some extra visual weight over the default and primary action buttons. These buttons should be used sparingly. Remember: when everything's important, nothing's important. To create an accent button, add the btn--accent class alongside btn.

Sizes

To change the size of a button, add the btn--small or btn--large class alongside btn. The default size is regular.

Widths

The width-modifying class changes the button to be relative to the width of its container. To create a button that takes up the full width of its container, add the btn--full class alongside btn. To create a button that takes up half the width of its container, add the btn--half class alongside btn.

Inverse

Inverse buttons are to be used on special-format pages that have black or dark backgrounds. To create an inverse button, add the btn--inverse class alongside btn.

Icons

To give more meaning to a button, icons can be used alongside button text. A button can simply be an icon as well. There are a variety of classnames that correspond with icons. Please see the icon page for reference.
